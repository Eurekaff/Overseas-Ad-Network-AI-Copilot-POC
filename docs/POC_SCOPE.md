# POC Scope

## 1. 第一版目标

第一版 POC 聚焦 5 分钟面试主线展示，覆盖三个高质量业务流程，而不是堆叠大量浅功能。

核心目标是展示完整产品闭环：

异常发现 -> 指标拆解 -> AI 解释 -> 建议生成 -> 人工确认 -> 后续观察。

## 2. 第一版包含内容

### 广告主投放看板 + AI 异常解释

业务案例：海外手游广告主发现美国市场近 7 日 CPA 上升、ROAS 下降。

系统需要展示：

- 消耗
- 展示量
- 点击量
- CTR
- CVR
- CPA
- ROAS
- 预算利用率
- 地区表现
- 素材表现

AI 需要输出：

- 异常摘要
- 关键证据
- 可能原因
- 优化建议
- 优先级
- 风险提示
- 后续观察指标

### 开发者收益看板 + AI 收益波动分析

业务案例：海外休闲游戏开发者发现巴西地区激励视频广告收入下降。

系统需要展示：

- 请求量
- 填充率
- 展示率
- eCPM
- 广告收入
- 加载耗时
- 广告源表现
- 国家表现

AI 需要输出：

- 收入下降原因
- 影响最大的广告位 / 国家 / 广告源
- 指标拆解
- 配置建议
- 风险提示
- 后续观察指标

### 内部运营工作台 + AI 报告辅助

业务案例：运营人员基于广告主或开发者异常，一键生成复盘与对外沟通材料。

支持任务模板：

- 生成客户周报
- 生成异常复盘
- 生成对外沟通话术
- 总结工单并给出处理建议

同时提供自由输入入口，但不作为主入口。

## 3. 第一版不包含内容

- 真实广告投放
- 真实广告平台 API
- 真实广告 SDK 接入
- 真实归因系统
- 真实结算系统
- 复杂登录注册
- 多租户权限
- 自动预算调整
- 自动广告源配置
- 复杂算法训练
- 完整广告投放后台
- 完整 CRM 或工单系统

## 4. 角色范围

第一版通过角色切换模拟三类用户：

- Advertiser：广告主
- Developer：开发者
- Operator：内部运营 / 销售 / 客户成功

不做真实账户权限校验。

## 5. 语言范围

第一版目标是完整双语：

- 导航
- 页面标题
- 表单字段
- 指标名称
- 图表字段
- AI 报告
- 建议动作
- 风险提示
- 后续观察指标

中英文内容可以基于同一份结构化数据渲染。

## 6. LLM 范围

第一版开发时可以先使用 mock LLM 输出。

系统设计需要预留 Qwen 接入：

- 独立 provider 层。
- 明确输入 schema。
- 明确输出 schema。
- mock 与真实 provider 可切换。

## 7. 闭环范围

第一版需要展示轻量闭环：

- 用户查看 AI 建议。
- 用户标记建议状态：采纳、暂不采纳、需要人工复核。
- 系统展示后续观察指标。
- 使用模拟数据展示采纳后的 24h / 7d 指标变化。
