# User Scenarios

## 1. 角色总览

本 POC 面向三类核心角色：

- 广告主：关注投放效果和预算效率。
- 开发者：关注广告变现收入和广告位表现。
- 内部运营 / 销售 / 客户成功：关注客户服务效率、异常解释和沟通材料生成。

## 2. 广告主场景

### 2.1 用户画像

广告主是一家海外手游发行团队，正在通过广告联盟投放买量，目标市场包括美国、巴西、印度尼西亚等国家。

核心目标：

- 控制 CPA。
- 提升 ROAS。
- 保持预算消耗稳定。
- 快速发现素材疲劳和转化异常。

### 2.2 核心问题

广告主发现：

- 近 7 日 CPA 明显上升。
- ROAS 下降。
- 美国市场表现变差。
- 部分素材点击率稳定但转化率下降。

### 2.3 用户任务

用户进入广告主诊断模块，选择：

- 广告账户：Galaxy Quest Studio
- 广告计划：US RPG User Acquisition
- 时间范围：近 7 日 vs 前 7 日
- 问题类型：CPA 上升 / ROAS 下降

系统需要帮助用户回答：

- CPA 为什么上升？
- 是 CTR 问题、CVR 问题，还是成本问题？
- 哪个国家、广告组或素材影响最大？
- 下一步应该优先检查什么？

### 2.4 期望输出

系统输出：

- 近 7 日 CPA 上升幅度。
- ROAS 下降幅度。
- 主要影响地区。
- 主要影响素材。
- 异常证据。
- 优化建议。
- 风险提示。
- 后续观察指标。

## 3. 开发者场景

### 3.1 用户画像

开发者是一家海外休闲游戏工作室，通过激励视频和插屏广告变现，主要流量来自巴西、墨西哥、印度尼西亚和美国。

核心目标：

- 提高广告收入。
- 保持填充率稳定。
- 提高 eCPM。
- 降低广告加载耗时。
- 发现广告源或广告位异常。

### 3.2 核心问题

开发者发现：

- 昨日广告收入下降。
- 巴西地区激励视频收入下降明显。
- 请求量基本稳定，但填充率下降。
- 某广告源返回率异常降低。

### 3.3 用户任务

用户进入开发者诊断模块，选择：

- App：Puzzle Island
- 广告位：Rewarded Video - Level Complete
- 国家：Brazil
- 广告源：All Sources
- 时间范围：昨日 vs 前日

系统需要帮助用户回答：

- 收入下降是由请求量、填充率、展示率还是 eCPM 导致？
- 哪个广告位、国家或广告源影响最大？
- 是否需要调整瀑布流或备用广告源？
- 后续应该观察什么指标？

### 3.4 期望输出

系统输出：

- 收入下降幅度。
- 填充率变化。
- eCPM 变化。
- 影响最大的广告源。
- 配置建议。
- 风险提示。
- 24 小时观察指标。

## 4. 内部运营 Copilot 场景

### 4.1 用户画像

内部运营 / 销售 / 客户成功团队需要同时服务多个广告主和开发者客户。团队日常需要解释异常、生成周报、准备客户会议材料和撰写对外沟通文案。

核心目标：

- 降低报告制作时间。
- 提高异常解释一致性。
- 帮助新人快速理解客户背景。
- 快速生成中英文沟通材料。

### 4.2 核心任务模板

第一版支持以下任务：

- 生成客户周报。
- 生成异常复盘。
- 生成对外沟通话术。
- 总结工单并给出处理建议。

### 4.3 用户任务

运营人员选择任务模板：

生成 Galaxy Quest Studio 本周投放复盘。

系统自动引用广告主诊断数据，输出：

- 客户画像。
- 本周核心指标变化。
- 异常原因。
- 建议动作。
- 对外沟通话术。
- 内部备注。
- 风险提醒。

### 4.4 期望输出

输出内容需要可直接用于客户沟通，但也要区分内部备注和外部话术，避免将不确定判断直接暴露给客户。

## 5. 10 分钟演示路径

建议演示顺序：

1. 进入平台首页，说明这是海外广告联盟 AI 诊断与提效平台。
2. 切换到广告主角色，演示 CPA 上升诊断。
3. 展示 AI 输出不是空泛结论，而是包含证据和建议。
4. 标记建议状态，展示后续观察指标。
5. 切换到开发者角色，演示激励视频收入下降诊断。
6. 切换到内部运营角色，用任务模板生成客户复盘和对外话术。
7. 切换语言，展示完整双语能力。
8. 总结产品闭环和后续 Qwen 接入设计。
