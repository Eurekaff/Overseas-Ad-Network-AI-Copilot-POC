# Overseas Ad Network Embedded AI Assistant POC

中文名称：海外广告联盟业务提效 POC：嵌入式 AI 辅助诊断

## 1. 项目定位

本项目是一个面向海外广告联盟真实业务流程的轻量 POC，用于展示广告联盟业务理解与嵌入式 AI 辅助能力设计。

项目不搭建独立 AI 平台，不追求真实上线，也不接入真实广告平台、真实 SDK 或真实结算系统。它通过模拟数据展示 AI 如何嵌入广告主看板、开发者收益看板和内部运营工作台，辅助解释、报告生成、沟通和后续跟踪。

## 2. 展示目标

主要展示对象是面试官，建议主线演示时长控制在 5 分钟左右。

本 POC 希望展示：

- 对海外广告联盟双边平台业务的理解。
- 对广告主投放效果诊断的指标拆解能力。
- 对开发者广告变现收益诊断的指标拆解能力。
- 对内部运营 / 销售 / 客户成功团队提效场景的产品化理解。
- 对 AI 不是独立平台或简单聊天框，而是嵌入业务流程的辅助能力的判断。
- 对“规则识别 + AI 解释 + 人工确认 + 后续观察”闭环的设计能力。

## 3. 业务背景

本 POC 的业务背景设定为：

- 广告主侧：海外手游广告主，关注安装、注册、付费、CPA、ROAS 和预算利用率。
- 开发者侧：海外 App / 游戏开发者，通过激励视频和插屏广告变现，关注请求量、填充率、展示率、eCPM、收入和加载耗时。
- 平台侧：广告联盟平台连接广告主预算和开发者流量，完成广告请求、匹配、展示、点击、转化、归因、结算和优化。

## 4. 产品愿景

让海外广告联盟中的关键角色在已有工作台内更快理解数据异常、更快定位原因、更快生成可执行建议，并形成可追踪的优化闭环。

AI 辅助的核心价值不是替代人做高风险决策，而是降低诊断和沟通成本，让用户从“看报表、猜原因、写报告”转向“看证据、审建议、跟结果”。

## 5. 核心闭环

POC 需要展示以下闭环：

1. 系统发现指标异常。
2. 用户进入结构化诊断场景。
3. 既有业务流程基于规则拆解指标变化。
4. AI 生成异常解释、建议动作和沟通内容。
5. 用户人工确认建议状态。
6. 系统展示后续观察指标和模拟改善结果。

## 6. 语言与产品形态

- 支持完整中英文双语切换。
- 不做真实登录，但提供角色切换。
- 产品风格为 B2B SaaS 业务工作台。
- 内部运营报告辅助以任务模板为主，自由输入为辅。

## 7. 技术方向预留

第一阶段可以使用 mock LLM 输出，保证演示稳定。

后续预留 Qwen 接入能力，建议以 provider 接口形式设计：

- MockProvider
- QwenProvider

第一版不要求立即完成真实 LLM 调用，但产品与数据结构应为后续接入真实模型留好扩展点。
